  Playability Issues and Fixes:


  Issue 1: Initial Player Mobility and Unexplained Restrictions
   1. What went wrong: The player starts with severely restricted movement and
      jump height, which contradicts the basic movement instructions. There's no
      in-game hint about why movement is restricted or how to resolve it, leading
      to immediate frustration and confusion.
   2. Root Cause: The playerSpeed and gravity are set to restrictive values
      (playerSpeed = 50, gravity.y = 800) in create(). While dragging staticWord
      resolves this, the player isn't informed of this initial state or the
      solution.
   3. Fix: Add a temporary on-screen hint at the start of the game explaining the
      player's initial state and guiding them towards the 'STATIC' keyword.


  Issue 2: Keyword Return Animation (Sentence Rule)
   1. What went wrong: When a keyword is dropped in empty space, it animates back
      to its drag start position, not its original sentence position. Also, the
      alpha resets immediately, removing the fading effect. The design doc states
      it should return to its "original sentence".
   2. Root Cause: The returnToOriginalPos function uses word.input.dragStartX and
      word.input.dragStartY instead of the keyword's original spawn coordinates.
      The onComplete callback immediately resets word.alpha = 1;.
   3. Fix:
       * Store originalX and originalY for each Keyword instance upon its
         creation.
       * Modify returnToOriginalPos to use word.originalX and word.originalY for
         the tween's target.
       * Remove the word.alpha = 1; from the onComplete of the tween. The alpha
         should return to 1 only when the word is picked up again or permanently
         removed. For now, we will leave it at alpha: 0.2 or fade it out
         completely. The description says "slowly fades and returns to its
         original sentence", so the fade should be persistent until grabbed
         again.

  Issue 3: No Visual Feedback for "Lava Neutralized!"
   1. What went wrong: When the 'WATER' keyword is used to neutralize 'lava', the
      floor color changes, and a console message appears, but there's no clear
      on-screen confirmation for the player.
   2. Root Cause: Feedback for neutralizing lava is only printed to the console.
   3. Fix: Display a temporary on-screen message like "Lava Neutralized!" when
      the 'WATER' keyword successfully removes the 'lava' property.
